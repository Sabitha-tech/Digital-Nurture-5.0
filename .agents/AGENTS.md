# Plagiarism-Free and Human-Like Code Rules

When generating code or solutions:
1. Ensure all code is customized and plagiarism-free. Do not copy exact solutions from online sources verbatim.
2. Use clean, natural, and standard variable/class names tailored to the context (avoid generic names like `MyClass` or `MyService` where possible, unless requested).
3. Do not include any AI markers, boilerplate comments, or "Generated by AI" signatures.
4. Keep the code style consistent with clean human development practices.
